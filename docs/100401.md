# LibreChat — Change Log (100401)

Worklog / Post 0.1.3 Improvements

---

## Friends UI & Layout

* Fixed multiple responsive layout issues in the Friends view:

  * Buttons and panels now behave correctly on smaller window sizes
* Improved Friends rail spacing:

  * Removed excess vertical gaps
  * Cleaned up inconsistent alignment
* Added search bar to Friends section
* Moved add-friend action:

  * Now a small icon button in the top-right of the Friends view
* Integrated custom add-friend icon assets:

  * Includes dark/light theme variants
* Fixed burger menu styling:

  * No longer appears as an unstyled white button
* Improved add-friend button theming to match client theme

---

## Notifications & Unread Logic

* Fixed Friends notification/unread state:

  * Now persists correctly when leaving and returning to the page
* Fixed unread glow logic:

  * Only triggers when the latest message is incoming
* Cleaned up notification color variable handling:

  * Better consistency across themes

---

## DMs & Chat Behavior

* Added first-pass message actions:

  * Reply
  * Edit
  * Delete
* Implemented within encrypted DM model:

  * Actions sent as encrypted payloads
* Fixed React crash:

  * Message bodies were sometimes objects instead of renderable text
* Improved encrypted DM UX:

  * Hid composer when chat is locked/unencrypted
  * Prevents layout breaking during lock screen
* Improved lock animation:

  * Closing animation now matches opening pose more closely
  * Reduced one lock timer from 5s → 3s
* Fixed remove-friend confirmation bug:

  * No longer breaks input focus after use

---

## Friends Code Structure

* Refactored Friends feature into a feature-based structure:

  * Moved to `Application/src/features/friends`
* Reduced size and complexity of `FriendsHome.jsx`
* Extracted:

  * Utility functions (e.g. message preview helpers)
  * Modal/confirmation components

---

## Per-Server Profile System

* Added backend support in `SelfHServer` for per-server profile descriptions
* Restricted editing:

  * Only authenticated user can edit their own description
* Added frontend actions:

  * Fetch and save server-specific profile descriptions
* Integrated into profile dock when inside a server

### UI Behavior

* Profile description is now:

  * Read-only by default
  * Editable via a small settings button

## Profile & Media Fixes

  * Adjusted `SelfHServer` auth verification to handle core auth host differences
* Fixed profile media loading behavior:

  * Own avatar/background no longer blocked by auto-load settings
* Improved profile readability:

  * Added frosted/blurred plate behind profile name
* Fixed profile dock layout:

  * Prevented clipping/overflow
  * Card now floats above chat pane correctly
