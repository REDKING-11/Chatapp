# LibreChat — Change Log (09-100401)

For 0.1.2 Alpha

---

## Identity & Profiles

* Added username tags/handles support (e.g. `REDKING_11#0011`, `MrHaamu11#6969`, `JohnDoe#0001`, `JaneDoe#0002`)
* Added profile display-name support to the core auth/user profile flow
* Added profile editing API with a new profile update endpoint

### Profile Media

* Added profile media settings for avatar and background uploads
* Moved avatar/background upload controls into Profile settings (removed from small profile dock)
* Added media limits and crop output rules:

  * Avatars output as **512 × 512**
  * Banners output as **1200 × 400**
  * Includes file-size checks and user-facing help text
* Added profile media manifest support through the self-host server
* Enabled public read support for shared profile assets
* Added profile media auto-load settings:

  * Avatars
  * Backgrounds
  * “Shared servers only” behavior

---

## Chat Identity & UX

* Added DM/group chat avatar display support with fallback initials
* Added minimal chat identity mode
* Implemented smart initials fallback for group chats where multiple users share the same first letter
* Added chat name display setting:

  * Use display names
  * Use username tags
  * Not both simultaneously

### Chat Layout & Styling

* Added message alignment settings:

  * Normal (others left / you right)
  * Everyone left
  * Everyone right
  * Reversed layout
* Reduced visual weight of timestamps and names for a cleaner, more minimalist chat style

---

## Messaging & Relay

* Added default offline relay window set to **24 hours**
* Fixed conversation settings relay-change request flow
* Fixed old-chat download request prompt appearing when no old chat/local access exists
* Fixed “Loaded X encrypted messages while you were away” banner:

  * Now only counts **newly imported messages**
* Fixed stale desktop notifications firing for already-read or old relay messages after refresh/reconnect
* Added seen/read timestamp baseline for existing conversations:

  * Prevents old messages from appearing unread on login

---

## Friends & Unread State

* Fixed Friends glow/unread state not clearing when:

  * Conversation is visibly open
  * But Electron focus state is inconsistent (e.g. DevTools open)

---

## Servers & Connectivity

* Added server online/offline health checking through Electron IPC
* Added periodic server recheck:

  * Allows offline servers to come back online without refreshing the client
* Added mid-session offline handling:

  * Prevents repeated requests to unavailable servers (channels/profile assets)
* Fixed profile asset manifest requests:

  * Prevented spamming/401 errors when server auth or offline state is invalid

---

## Client Stability

* Fixed React “setState while rendering” issue in client settings

  * Resolved by deferring settings-change broadcasts

---

## Build & System

* Added installer wizard configuration changes for Windows/MSI:

  * Improved directory picker behavior
  * Updated install folder naming under `RedFolder/LibreChat`
* Added documentation file `09-100401.md`
* Rebuilt frontend production assets in `Application/dist`
