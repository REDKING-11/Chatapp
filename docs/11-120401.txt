## Core Text And Content
- Added real `Pretext.js` support for text-like builder blocks, so text and heading nodes can use an arithmetic text layout engine instead of always depending on browser DOM measurement.
- Added broad Markdown support across builder content, chat messages, DMs, and related text surfaces.
- Added Markdown preview while typing, so likely Markdown shows a live rendered preview before sending.
- Expanded Markdown features to include headings, emphasis, code blocks, blockquotes, lists, task lists, tables, horizontal rules, links, and images.
- Added Mermaid diagram rendering in Markdown.
- Added simple chart rendering from Markdown code fences.
- Kept Markdown rendering sanitized rather than allowing arbitrary raw HTML/script embeds.

## Reactions And Message UX
- Added Discord-style reactions with quick emoji options and a fuller emoji picker.
- Moved reactions so they belong under the target message rather than acting like standalone visible messages.
- Reworked the reaction controls so message actions and reaction controls live under the message instead of stretching the message bubble awkwardly.
- Fixed multiple reaction-picker issues: sticky open state, outside-click close, picker centering, backdrop styling, and multi-click behavior.
- Fixed the deeper DM reaction pipeline so reactions rebuild into the target message after conversation reload instead of becoming blank “ghost” messages.
- Changed reaction chips to render under the actual message, while the footer holds controls.
- Added a centered overlay picker for reactions and later fixed its selection wiring so choosing emoji actually worked.
- Restored the picker and tightened its open/close logic after regressions.

## Message Composer
- Added normal emoji insertion into the composer, separate from reactions.
- Added basic file sending from the composer.
- Redesigned the composer layout so tools are smaller and better placed:
  - file button on the left
  - emoji trigger inside the field
  - smaller send button on the right
- Made the text area non-resizable so it stopped stretching the whole composer strangely.
- Restored the animated “dancing emoji” trigger and made it rotate to a different emoji every 20 seconds.
- Fixed the emoji trigger centering by reusing the same centering logic as the paperclip button.
- Fixed the composer emoji picker backdrop so it no longer turned the whole screen into a giant blue button.
- Changed the composer emoji picker so it stays open after selecting an emoji, making it easier to choose several in a row.

## File Sharing And Attachments
- First added lightweight file sharing by embedding small files into message text as Markdown/data URLs.
- Then replaced that for secure DMs/groups with a more real no-storage attachment flow:
  - sender sends attachment metadata, not a giant embedded message blob
  - recipient sees a download button
  - file transfer streams live
  - both sides can show progress
  - no long-term server storage is required
- Polished attachment cards with better file metadata, status pills, progress display, remove button alignment, and cleaner layout.
- Improved long filename handling and centered the remove `x`.
- Clarified that current attachment sharing is good for smaller files, while large mods/modpacks would need a more advanced hosted or peer-to-peer solution later.

## DM And Group Chat Systems
- Added disappearing messages as a separate DM setting from relay retention.
- Added agreement flow so both people must accept disappearing-message policy changes.
- Added more disappearing-message time options, including `2 months`, `4 months`, and `6 months`.
- Added a wheel-style picker for relay retention and disappearing-message settings instead of plain dropdowns.
- Fixed a DM sync issue where imported/opened conversations could overwrite local history instead of merging.
- Improved DM message rebuilding logic so control events like reactions are folded into target messages instead of showing as visible messages.
- Added first-pass no-storage live attachment transfer support for secure DMs/groups.
