# Major update to DM reliability and encrypted media handling

## LibreChat v0.1.5-alpha

> Standalone Windows builds available
> Installer and update system reworked

This update focuses on delivery reliability, update flow, encrypted media, networking security, and deep stability improvements across DMs, auth, and deployment.

---

## Highlights

* Reworked Windows install/update system (WiX/MSI, removed Squirrel)
* Added inline encrypted images inside DMs (Markdown-based)
* Improved DM delivery accuracy (live vs relay-aware states)
* Added recovery for broken encrypted storage (no full resets needed)
* Introduced better handling for missing DM keys and local data
* Enforced TLS-only networking across the client and services
* Fixed major issues introduced in v0.1.3 and v0.1.4 (auth, DM, realtime)

---

## Messaging & UX

* Delivery state now updates correctly:

  * `Sent securely` when at least one device receives the message live
  * `Offline relay` only for true relay-only delivery
* Delivery state persists across refresh/reload
* Inline encrypted images now render directly inside messages:

  * Markdown-positioned (`dm-embed://`)
  * Paste and file picker supported
* Added custom inline image sizing:

  * `{25x25}`, `{250pxx180px}`, etc.
* Improved handling for missing conversations:

  * Explicit states: `ready`, `missing-local`, `missing-key`
  * Composer disables when access is unavailable
* Background auto-import for missing conversations

---

## Inline Encrypted Media

* Small images (`png`, `jpeg`, `webp`, ≤ 256 KB) now embedded directly in encrypted DM payloads
* Larger files remain on the attachment/file-transfer path
* Message body now controls which embeds are sent
* Secure markdown rendering:

  * Remote images disabled
  * `<img>` injection blocked
* Stable rendering improvements:

  * Removed `blob:` lifecycle issues
  * Switched to `data:` URL rendering
  * Fixed React StrictMode remount issues
* Added structured diagnostics for embed parsing and rendering

---

## Security & Storage

* Auth token failures no longer crash startup
* Invalid encrypted auth data now triggers safe logout/reset
* Secure DM storage recovery:

  * Broken stores quarantined instead of wiping full app data
  * Legacy paths migrated (`Chatapp` → `LibreChat`)
* Session storage secured:

  * Tokens hashed (`SHA-256`) instead of stored raw
* Encrypted attachments fully contained in DM payloads

---

## Networking, Auth & Deployment

* Enforced TLS-only networking:

  * Rejects `http://` and `ws://`
  * Uses `https://` and `wss://` only
* Removed hardcoded localhost fallbacks
* Environment-driven connection handling restored
* Improved joined-server handling:

  * Invalid/insecure entries skipped instead of crashing UI
* Improved auth/session restoration:

  * Prevents stale token spam
  * Clearer error handling
* Improved realtime behavior:

  * Backoff reconnect logic
  * Reduced duplicate subscription spam
  * Automatic fallback to relay when realtime unavailable

---

## Core / Backend Fixes

* Fixed major auth mismatch:

  * Unified session validation (hashed tokens across all routes)
  * Eliminated `401 Invalid token` after login
* Fixed secure-DM device bundle handling:

  * Prevented mutation of signed bundle data
  * Restored signature verification reliability
* Fixed realtime deployment gaps:

  * Added missing `package.json`
  * Ensured dependency installation works correctly

---

## Trust Model & Device Handling

* Simplified trust model:

  * New devices trusted automatically after login
* Improved revoked-device behavior:

  * Revoked devices stay signed in but cannot use secure DMs
* Added MFA-based device re-authorization flow
* Added explicit backend device states:

  * `DEVICE_NOT_REGISTERED`
  * `DEVICE_REAUTH_REQUIRED`
* Improved client UX:

  * Clear device states (`Active`, `Current`, `Revoked`)

---

## Windows Installer & Updates

* Removed Squirrel update system
* Restored WiX/MSI installer flow
* Update handling now:

  * Installer / release-page based
  * No Squirrel-specific logic
* Cleaned Electron startup hooks and app-id handling

---

## Server & Deployment Stability

* Production deployment stabilized:

  * Nginx moved to `127.0.0.1:8080`
  * Realtime service on `127.0.0.1:3010`
  * Caddy handling HTTPS (`80/443`)
* Fixed:

  * Port conflicts (`:80`)
  * Certificate permission issues
  * Missing SNI handling for IP-based HTTPS
* Verified HTTPS proxy flow through Caddy
* Identified remaining gap:

  * PM2 startup persistence for realtime

---

## Stability & Fixes

* Fixed mixed live + relay delivery state issues
* Fixed delivery state reverting after reload
* Fixed inline image rendering issues (blob lifecycle, remounts)
* Fixed secure-DM IPC error noise for expected states
* Fixed crashes caused by invalid joined-server entries
* Fixed DM send failures caused by missing device-chain logic
* Fixed multiple regressions introduced in v0.1.3 / v0.1.4

---

## Developer & Diagnostics

* Added structured diagnostics system:

  * Inline image lifecycle
  * Embed parsing
  * DM round-trip validation
* Added debug-mode tracing and error categorization
* Expanded test coverage:

  * Delivery state
  * Inline embeds
  * Local conversation access
* Improved realtime deployment debugging:

  * `.env` handling
  * systemd config
  * database auth setup

---

## Notes

* Inline encrypted images currently limited to small files (≤ 256 KB)
* Large file sharing still relies on attachment system
* Some realtime and multi-device flows still need broader live validation
* PM2 auto-start for realtime service may require additional setup
