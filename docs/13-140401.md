# Chatapp — Update Log

LibreChat 0.1.4
---

## Overview

This update focuses on improving how direct messages behave as a real messaging system, strengthening end-to-end encryption boundaries, improving authentication and session handling, and fixing several core UX and login issues.

The work primarily covered:

* DM settings and UI behavior
* Messaging reliability and delivery clarity
* Security and encryption boundaries
* Authentication, sessions, and MFA
* Backend ordering and data handling improvements

---

## DM Settings & UI

The DM settings interface was heavily reworked after earlier implementations proved unstable.

* Rebuilt the relay and disappearing-message picker:

  * Replaced the unstable 3D-style selector with a simpler, stable picker
  * Fixed expansion issues where the full list would incorrectly render
  * Limited the UI to a clean 3-option selector
* Removed unnecessary UI elements:

  * Removed visible up/down arrows from the picker
* Fixed default state handling:

  * New or unset conversations now correctly display **“24 hours”** instead of “No relay”
* Improved interaction handling:

  * Fixed passive wheel listener warnings by moving wheel logic outside React passive handling
  * Preserved mouse wheel and keyboard navigation support

### Additional UI changes

* Moved **Logout** into settings
* Added a **Delete account** button (currently non-functional, shows WIP state)
* Removed top-level **Join Server** button from the main layout

## DM Behavior & Inbox Experience

Direct messages were reworked to behave more like a real messaging inbox instead of a static contact list.

### Backend ordering improvements

* Changed friend list ordering logic:

  * Old: sorted by `friendship.created_at`
  * New: sorted by activity using:

    * `conversation.updated_at`
    * fallback → `responded_at`
    * fallback → `created_at`
* Result:

  * Most recently active conversations appear first

### Frontend behavior

* Added client-side sorting for immediate responsiveness:

  * Uses preview timestamps to reorder without waiting for backend refresh
* Fixed refresh behavior:

  * No longer resets selected conversation
  * Preserves scroll and ordering
  * Only refetches when relevant conversation data changes

## Message Delivery State

Added a full **per-message delivery state system** for outgoing messages.

### Delivery states

Each outgoing message now shows:

* **Sent securely**
* **Offline relay**
* **Failed**

### How it works

* Message send returns an `outboundMessageId`
* A new state map tracks delivery per message:

  * `messageDeliveryById`
* Realtime events update delivery status:

  * Dropped recipients → `failed`
  * Offline recipients → `queued`
  * Otherwise → `sent`
* Messages default to `sent` after successful send unless updated by realtime events

### UI

* Added compact delivery badge next to message metadata:

  * Green → Sent
  * Amber → Relay queued
  * Red → Failed
* Only shown on outgoing messages

### Important limitation

* This is **not a read receipt system**
* It does **not confirm**:

  * Message seen
  * Message decrypted
  * Message opened
* Only reflects known delivery state from sender perspective

## Security & E2EE Improvements

A large portion of this update focused on strengthening security boundaries.

### Key isolation

* Moved DM private keys fully into Electron main process
* Blocked secret-shaped IPC responses
* Prevented renderer access to sensitive key material

### Storage security

* Encrypted DM storage using `safeStorage`
* Prevented plaintext leaks in export paths

### Transport enforcement

* Enforced HTTPS/WSS for remote endpoints
* Allowed HTTP only for localhost development

### Device & identity security

* Added:

  * Device fingerprints / safety numbers
  * Trusted device system
  * Device approval flow
  * Device revocation support
* Added signed device bundles with verification

### Cryptographic improvements

* Added:

  * Ed25519 message signatures
  * Replay protection
  * Key rotation and rekeying
* Ensured all control events are encrypted:

  * Edits
  * Deletes
  * Reactions
  * Disappearing-message updates

### Attachments

* Encrypted attachments before transfer
* Embedded attachment keys inside encrypted DM envelopes

### Relay system

* Introduced ciphertext-only DM relay store
* Reduced metadata retention
* Shortened retention options

Authentication was refactored and significantly improved.

### Login & identity fixes

* Login now supports:

  * `username`
  * `username#tag`
* Fixed password handling:

  * Removed `.trim()` behavior (no silent password modification)
* Fixed registration:

  * Proper username/tag parsing

### MFA & sessions

* Added:

  * MFA (TOTP)
  * MFA setup / enable / disable flows
  * MFA challenge handling
* Added session system:

  * Session listing
  * Session revocation
  * Session naming (`X-Chatapp-Session-Name`)

### Backend improvements

* Refactored auth into shared helpers
* Fixed DB handling (including port support)
* Fixed CORS issues introduced by new headers
* Added schema compatibility handling:

  * Prevents partially upgraded DBs from breaking MFA

## Session Security (Token Hashing)

Session storage was redesigned to eliminate raw token exposure.

### Changes

* Added `token_hash` column to sessions table
* Tokens are now:

  * Hashed using SHA-256 before storage
  * Stored as `sha256:<hash>` marker instead of raw token

### Behavior

* Incoming requests:

  * Token is hashed before lookup
* Existing sessions:

  * Automatically migrated
  * Raw tokens replaced with non-replayable markers

### Result

* Database no longer contains usable session tokens
* Reduces risk from DB leaks or admin misuse
